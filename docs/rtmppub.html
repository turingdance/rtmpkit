NAME

    rtmppub − RTMP streaming media publisher

SYNOPSIS

    rtmppub −i input_file −r rtmp_url
    rtmppub −h

DESCRIPTION

    rtmppub is a tool for publishing media content to RTMP servers.

    rtmppub reads media from a file and pushes it to the specified RTMP server. The input can be:

    • An FLV file (.flv)
    • An H264 file (.h264 or .264)

    The RTMP URL should be of the form 

      rtmp://hostname[:port]/app/playpath

OPTIONS

Required Parameters

    These options are required for all operations.

    −−input −i input_file
    Input file path. Can be:
    • A file path ending with .flv for FLV files
    • A file path ending with .h264 or .264 for H264 files

    −−rtmp −r url
    URL of the RTMP server. E.g., rtmp://127.0.0.1:1935/live/test

Miscellaneous

    −−help −h
    Print a summary of command options.

INPUT MODES

    FLV File Mode

      When the input is a file ending with .flv, rtmppub reads the FLV file and pushes it to the RTMP server. It parses FLV tags (audio, video, script) and sends them with proper timestamps.

      Supported tag types:

      • Audio (0x08)
      • Video (0x09)
      • Script/Data (0x12)

    H264 File Mode

      When the input is a file ending with .h264 or .264, rtmppub reads the raw H264 stream in Annex B format. It automatically detects SPS/PPS and IDR frames, constructs FLV video tags, and sends them to the server.

      Features:

      • Automatically detects NALU start codes (00 00 00 01)
      • Parses SPS to extract video dimensions
      • Generates AMF MetaData with video info
      • Sends at 25fps by default

EXAMPLES

    Push FLV file: 
      rtmppub -i input.flv -r rtmp://127.0.0.1:1935/live/test

    Push H264 file: 
      rtmppub -i input.h264 -r rtmp://127.0.0.1:1935/live/test

DEPENDENCIES

    librtmp
    RTMP library for communication with RTMP servers.

COMPILATION

      make rtmppub
    This will compile rtmppub.c and link against librtmp.a statically.
